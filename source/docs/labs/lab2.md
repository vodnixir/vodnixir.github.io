# Лабораторная работа 2. Основы NumPy: массивы и векторные операции

## Описание задачи

Цель лабораторной работы — изучить основы библиотеки NumPy, научиться работать с одномерными и многомерными массивами, выполнять векторные и матричные операции, а также применять базовые методы анализа и визуализации данных.

В ходе работы требовалось реализовать набор функций для:

- создания массивов и матриц;
- изменения формы массивов;
- транспонирования матриц;
- выполнения векторных операций;
- выполнения матричных вычислений;
- загрузки данных из CSV-файла;
- статистического анализа данных;
- нормализации значений;
- построения графиков.

Дополнительно код должен был:

- проходить тесты;
- содержать аннотации типов;
- содержать docstring;
- соответствовать требованиям PEP-8.

## Как была решена задача

Для выполнения лабораторной работы были использованы следующие библиотеки:

- `NumPy` — для работы с массивами, векторами и матрицами;
- `pandas` — для загрузки CSV-файла;
- `matplotlib` и `seaborn` — для визуализации данных;
- `pytest` — для тестирования.

### 1. Создание и преобразование массивов

Сначала были реализованы функции для базовой работы с массивами:

- создание одномерного массива чисел от 0 до 9;
- создание матрицы размером 5x5 со случайными значениями;
- преобразование вектора формы `(10,)` в матрицу формы `(2, 5)`;
- транспонирование матрицы.

Эти операции позволяют понять базовую модель работы NumPy: данные хранятся в массивах, а большинство преобразований выполняются встроенными методами без циклов.

### 2. Векторные операции

Далее были реализованы функции для выполнения основных операций над векторами:

- поэлементное сложение;
- умножение вектора на скаляр;
- поэлементное умножение;
- скалярное произведение.

При решении использовалась векторизация NumPy, то есть операции выполнялись сразу над всем массивом. Это делает код короче, понятнее и эффективнее по сравнению с ручной обработкой элементов через `for`.

### 3. Матричные вычисления

После этого были реализованы функции для работы с матрицами:

- матричное умножение;
- вычисление определителя;
- вычисление обратной матрицы;
- решение системы линейных уравнений.

Для этих задач использовались функции из модуля `numpy.linalg`, так как он содержит готовые и оптимизированные алгоритмы линейной алгебры.

### 4. Работа с данными

Отдельная часть лабораторной работы была посвящена загрузке и обработке данных из CSV-файла. Данные считывались с помощью `pandas`, после чего преобразовывались в `NumPy`-массив.

После загрузки был реализован статистический анализ данных. Для массива вычислялись:

- среднее значение;
- медиана;
- стандартное отклонение;
- минимальное и максимальное значение;
- 25-й и 75-й перцентили.

### 5. Нормализация данных

Также была реализована min-max нормализация данных по формуле:

`(x - min) / (max - min)`

Такой способ позволяет привести значения к диапазону от 0 до 1. Это полезно при анализе данных и часто используется в задачах машинного обучения.

### 6. Визуализация

В последней части работы были реализованы функции построения графиков:

- гистограмма;
- тепловая карта;
- линейный график.

Графики сохраняются в отдельную папку `plots`, что позволяет использовать их в дальнейшем для анализа или включения в отчёт.

## Нюансы решения

### Векторизация вместо циклов

Одна из главных особенностей NumPy заключается в том, что многие операции можно выполнять без циклов. Например, сложение двух массивов делается напрямую через `a + b`, а не через перебор элементов.

Это делает программу не только короче, но и быстрее.

### Разница между поэлементным и матричным умножением

При выполнении работы важно было различать два разных типа умножения:

- `a * b` — поэлементное умножение;
- `a @ b` — матричное умножение.

Это разные операции, и они применяются в разных задачах.

### Использование reshape

Функция `reshape` не изменяет значения массива, а только задаёт новое представление данных в другой форме. Это удобно, когда нужно перейти от вектора к матрице или наоборот.

### Использование numpy.linalg

Для вычисления определителя, обратной матрицы и решения системы линейных уравнений удобнее использовать готовые функции `numpy.linalg`, чем реализовывать эти алгоритмы вручную.

### Особенность нормализации

При min-max нормализации важно помнить, что если все значения массива одинаковые, то выражение `(max - min)` станет равно нулю. В таком случае может возникнуть деление на ноль. Для обычных входных данных функция работает корректно, но этот случай стоит учитывать отдельно.

### Визуализация и сохранение графиков

При построении графиков важно не только создать рисунок, но и корректно сохранить его в файл. После сохранения нужно закрывать фигуру с помощью `plt.close()`, чтобы не накапливать объекты графики в памяти.
